# Compile ルール仕様書(エンジン実装用)

出典: COMP-MN01 Rulesheet (Updated) + The Compile Codex (rev. 2025-09-22)
対象: Main 1 + Aux 1。Phase 0 成果物。

## 1. ゾーンとオブジェクト

| ゾーン | 内容 | 公開情報 |
|---|---|---|
| field | 3ライン × 両プレイヤーのスタック | 表向き=公開 / 裏向き=持ち主のみ閲覧可 |
| deck | 各18枚(3プロトコル×6) | 枚数のみ公開 |
| hand | 手札 | 枚数のみ公開 |
| trash | 捨て札。**常に表向き** | 完全公開 |
| protocols | 各プレイヤー3枚。位置(左中右)を持ち、compiled フラグを持つ | 公開 |
| control | コントロールコンポーネント: neutral / p1 / p2 | 公開 |

### カードインスタンスの状態
- `faceUp: boolean` — 裏向きカードの値は **2**(常在効果で変更されうる)
- `owner` — 所有者。リコンパイル等で変わり、以後ゲーム終了まで(再変更まで)維持。trash/deck に行っても維持
- `committed` — ゾーン間移動中の遷移状態(§5)

### スタックの値
スタック値 = 各カード値の合計。カード値 = (表: 印刷値 / 裏: 2) に常在効果(upper)の修正を毎回適用して算出する(キャッシュせず都度評価)。

## 2. セットアップ

1. ドラフト: 先手1枚 → 後手2枚 → 先手2枚 → 後手1枚。各自のプロトコルはドラフト順に左から配置
2. 各自、自分の3プロトコルのカード6枚ずつ計18枚をシャッフルしてデッキに
3. 各自5枚ドロー。先手 = 最初にドラフトしたプレイヤー
4. control は neutral(入門モードでは control 無効化オプション)

## 3. ターン構造(フェイズ)

```
Start → Check Control → Check Compile → Action → Check Cache → End
```

1. **Start**: フェイズ開始時点で自分側の field 上に「見えている」`Start:` コマンドを**記録(note)**し、好きな順に1つずつ解決。解決前に field を離れた/見えなくなったコマンドは実行されない。フェイズ開始後に現れた `Start:` は発動しない
2. **Check Control**: 2ライン以上で相手より合計値が高ければ control を獲得(neutral からでも相手からでも)
3. **Check Compile**: いずれかのラインで「自スタック値 ≥ 10 かつ 相手スタック値より大」なら**強制的に**コンパイル(§4)。複数該当時は1ラインを選択。コンパイルした場合 Action はスキップ(コンパイルがそのターンの唯一のアクション)
4. **Action**: 「カード1枚プレイ」or「リフレッシュ」。プレイできるカードがなければリフレッシュ**必須**。コンパイルもドローもプレイもできない場合はスキップ
5. **Check Cache**: 手札が6枚以上なら5枚になるまで捨てる(= Clear Cache。これは discard)
6. **End**: Start と同じ記録方式で `End:` コマンドを解決

## 4. アクション詳細

### プレイ
- 表向き: **一致するプロトコルのライン**にのみ(相手側プロトコルのラインでも自分側スタックに置ける点に注意 — 一致が条件)。middle コマンドが発動
- 裏向き: 任意のライン。効果は発動しない
- スタックに既にカードがあれば新カードが一番上(uncovered)になり、下のカードを「覆う(cover)」

### リフレッシュ
- 手札が **5枚未満** のときのみ可能。5枚になるまでドロー
- control 保持時: まず control を neutral に戻し、どちらか一方のプレイヤーのプロトコルを並べ替えてよい(任意。やらなくても control は戻る)。その後ドロー

### コンパイル手順(順序厳守)
1. control 保持時: control を neutral に戻し、一方のプレイヤーのプロトコルを並べ替えてよい
2. 対象ラインの**両プレイヤーの全カード**を同時に削除(「all」効果 — uncover トリガーも middle も発動しない)。各カードは持ち主の trash へ
3. 自分のプロトコルを「Compiled」面に。**既に Compiled なら(リコンパイル)**: 代わりに相手のデッキトップを1枚引き、そのカードの所有権を得る
4. 開始後にスタック値が10未満/相手以下になっても続行する
5. 1ターンに1ラインのみ

### 勝利
自分の3プロトコルすべてが Compiled になったら勝ち(フリップした瞬間に判定)。

## 5. テキスト解決モデル(エンジンの核)

### コマンド3種
| 位置 | 種別 | 有効条件 |
|---|---|---|
| upper (Top) | 常在(Persistent) | 表向きである限り有効。**覆われていても有効** |
| middle (Middle) | 即時(Immediate) | プレイ/表返し/uncover でテキストが場に入った時に解決。**覆われた状態で表返されても発動しない** |
| lower (Bottom) | 補助(Auxiliary, 多くはトリガー) | 表向き **かつ** uncovered の時のみ有効 |

### 解決の基本
- テキスト場入り(play / flip-up / uncover)時、**割り込み(LIFO)** で解決する。解決中に別のテキストが場に入ればそちらを先に解決(Last in, first out)
- テキストの**所有者(カードのowner)**が解決方法(対象選択等)を決める
- 同時に複数トリガー → **手番プレイヤー**が解決順を決める
- 「After X:」トリガーは X とその結果がすべて解決し終わってから発動
- 効果はコストではない。可能な限り解決し、不可能な部分はスキップしてもカードはプレイされる。「〜した場合(If you do)」は前提行動が完了していなければ後段は発動しない
- 文(センテンス)ごとに独立して解決
- 「〜してもよい(may)」以外は強制。ゲーム状態を変更できるなら変更しなければならない(例: Water 4 が唯一の対象なら自身を戻す)
- 有効中だったテキストが覆われる/裏返る/場を離れると、残りのテキストは即座に無効(例外: 「this card」参照は維持 §5.4)

### 5.1 対象選択
- デフォルト: 両プレイヤーの **uncovered** カードのみ対象可(flip/delete/return/shift/reveal)
- 「あなたの/相手の」で所有側を限定
- 「覆われている(covered)」指定で covered を対象化
- 「**all(すべての)**」= covered 含む該当カード全部を**同時に**処理(個別トリガーなし)
- 「**each(それぞれの)**」= 対象を記録してから持ち主が選んだ順に**1枚ずつ**処理(各処理の結果を解決してから次へ)。covered は対象外(明記なき限り)
- 「**this card / that card(このカード/そのカード)**」= 対象指定不要の直接参照。covered でも操作できる。参照先が場を離れても参照は維持(Light 0 ルーリング: trash では表向き値で参照)
- スタックにないカード(hand/deck/trash)は flip/delete/shift 不可

### 5.2 キーワード
- **Draw X**: デッキ上から X 枚まとめて手札へ。1枚ずつではなく**バッチ**(トリガーはX枚引き終わってから)。デッキ不足時は trash をシャッフルして新デッキを作り続行。**リシャッフルはドローのみが誘発** — デッキ0枚でのデッキトップ参照系(discard/play/reveal the top)は何もしない
- **Discard X**: 手札から持ち主が X 枚選び trash へ(バッチ)。デッキトップを discard する効果も「discard」トリガーを誘発する
- **Delete**: field → trash
- **Return**: field → 持ち主の手札
- **Flip**: 表⇔裏。表返しで uncovered なら middle 発動
- **Shift**: 同じ側の別ラインへ移動。移動元の下のカードが uncover → そのトリガーを解決してから shift カードが着地(その間 shift カードは committed)
- **Reveal**: 情報を公開して元の状態に戻す(状態変更なし)
- **Rearrange**: プロトコル位置の入れ替え。**必ず変化させる**(同一配置は不可)。ライン上のカードは動かない。サイドはまたげない
- **Refresh(効果による)**: 通常のリフレッシュと同様(control 消費も含む — Spirit 0)
- **Clear Cache**: 手札5枚まで discard

### 5.3 Committed(ゾーン遷移中)
- ゾーン間を移動するカードは現ゾーンを離れて新ゾーンに「committed」になる
- 離脱の結果(uncover トリガー等)をすべて解決してから着地する
- committed 中: 向き・行き先を維持し、**いかなる効果の対象にもならない**。着地は妨害不可
- 「覆われる直前: まず〜(When this card would be covered: first...)」は committed カード着地前に解決。その効果での対象選択に committed カードは含められない
- 複数 committed はコミット順に1枚ずつ着地

### 5.4 個別ルーリング(Main 1 + Aux 1)
- **Death 0 / Life 0 / Water 1 / Chaos 0 型(each line)**: 対象ラインを記録 → 持ち主が選んだ順に1ラインずつ処理し、各結果を解決してから次へ。処理中に発生源が覆われたら中断(Life 0)
- **Gravity 2**: 「that card」参照なので flip 後に覆われていても shift する
- **Light 0**: flip → そのトリガー解決 → **現在の**値を参照してドロー(場外なら trash=表向き値6 / deck=裏扱い値2)
- **Light 3**: 裏向きカード群の shift は相対位置を維持して同一ラインへ
- **Metal 6**: top command により覆われていても自己削除。flip 効果で消えた場合その flip は消費済み(他に振り直せない)
- **Speed 2**: コンパイルの全削除を**自身のみ** shift で置換できる(コンパイル自体は進行)
- **Plague 3**: 「each other uncovered face-up card」— 記録して1枚ずつ flip、各結果を解決してから次へ
- **Hate 2**: 自分の最高値→相手の最高値の順。タイは選択。Hate 2 自身が最高値なら自己削除し、第2文は消滅
- **Metal 1**: 相手は次のターン、コンパイルアクションを取れない(テキストが見えている場合)

### 5.5 エラッタ(データ化時に反映必須)
| カード | 正しいテキスト |
|---|---|
| Death 1 (top) | Start: カードを1枚引いてもよい。そうした場合、他のカードを1枚削除する。その後、このカードを削除する |
| Fire 0 (bottom) | このカードが覆われる直前: まずカードを1枚引く。その後、他のカードを1枚反転させる |
| Life 0 (top) | End: このカードが覆われている場合、このカードを削除する(bottom コマンドは無し) |
| Metal 1 (middle) | カードを2枚引く。相手は次の相手ターンにコンパイルできない |
| Plague 3 (middle) | 他の覆われていない表向きカードをそれぞれ反転させる |
| Spirit 1 (top) | あなたがカードを表向きでプレイするとき、プロトコルが一致していなくてもよい |
| Hate 2 (middle) | あなたの最も値の高い覆われていないカードを削除する。相手の最も値の高い覆われていないカードを削除する |

## 6. 情報秘匿(オンライン化時の要件)

- デッキ順・内容: 秘密(枚数は公開)
- 手札: 本人のみ(枚数は公開)
- field の裏向きカード: **持ち主のみ**面を見られる。デッキトップを裏でプレイする場合、着地するまで見られない
- trash: 全公開(表向き)
